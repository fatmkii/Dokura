# Dokura服务端需求

本文件应与[共通需求](./PRD.md)一并阅读。Android客户端的专属需求见[客户端需求](./PRD-client.md)。

## 后台服务

### 服务范围

 - 服务端不需要实现从外网下载同人志资源。

### 状态模型

   - 文件分析状态固定为：`waiting_stable`（等待文件写入完成）、`analyzing`（正在分析）、`ready`（可用）、`retry_wait`（等待重试）、`failed`（处理失败）和`no_valid_content`（无有效内容）。外部删除的文件不保留可见状态，直接从列表隐藏。已有可用元数据的ZIP发生修改时，旧内容保持`ready`可展示，同时附加“正在重新分析”任务状态；新分析成功后原子替换。
   - 文件分析状态与封面缩略图状态分开。每个ZIP只保存一张封面缩略图，状态固定为`not_generated`、`queued`、`generating`、`complete`和`generation_failed`。API返回稳定的英文状态码，Android和Web显示对应中文文案。

### 文件发现与同步

#### 启动与重新扫描

   - Web服务和API在配置及元数据存储加载完成后立即可用，不等待启动扫描完成；启动扫描作为后台任务运行。扫描期间展示已有元数据并提示“正在扫描，列表可能变化”。新发现文件在单个ZIP分析完成后即可进入列表，已确认外部删除的文件立即隐藏；尚未重新验证的旧记录继续显示，实际读取时文件不存在则返回“文件不可用”，并由本轮扫描最终隐藏。
   - 服务启动时执行一次增量扫描，运行期间监听Content目录中的新增、删除、移动、重命名和修改事件。Web设置页提供“重新扫描文件”按钮，副文案为“扫描Content并补齐未处理的文件变化”；同一时间最多运行一个扫描，扫描中再次点击只标记一次“完成后再扫描”，重复点击不继续累加。
   - “重新扫描文件”完整遍历Content并与已有元数据增量对账，处理新增、漏掉的移动或重命名、修改和删除，将尚未入队的`waiting_stable`文件重新加入稳定性检查，并重新解析解析器版本过旧的记录。它不重试文件未变化的`failed`记录，也不重新生成已完成且仍有效的封面缩略图。

#### 存储不可访问时的处理

   - 只有成功遍历的目录范围才能根据“未发现”判定文件已删除。目录打开、读取或`stat`失败时记录该目录扫描失败，不删除其中已有记录；旧记录继续显示并附加“存储暂时不可访问”状态，实际读取失败按既有错误处理。扫描继续处理其他可访问目录，结果显示成功和失败目录数量及简短原因，详细路径和异常写入日志。后续监听事件或扫描成功访问后清除该附加状态并正常对账。
   - Content根目录无法访问时本轮扫描整体失败，保留全部已有记录，不推断任何删除。
   - Web显示扫描状态、开始时间、已检查目录项数量、发现变化数量和最近一次完成结果。扫描失败时记录错误并结束本轮；如果已标记“完成后再扫描”，仍启动下一轮。浏览器关闭或登录会话结束不取消已开始的扫描。
   - Content根目录不可访问时，Web和Android文件列表返回空列表；某个子目录不可访问时，该目录返回空列表，不显示特殊的“存储不可访问”文件项。访问失败只临时隐藏已有记录，不自动删除元数据、UUID、评分或封面缩略图；其他可访问目录继续扫描。NAS恢复后，相同路径和文件身份重新显示原记录并保留UUID和评分。用户主动执行“清理缓存”时仍以当时可访问的文件系统状态和确认统计为准。
   - 健康检查区分“服务可用”和“扫描完成”两个状态；Docker健康状态只依赖服务可用，扫描进行中不判定为不健康。

#### 文件监听范围

   - 首版完全忽略Content根目录内部的文件符号链接和目录符号链接。扫描器和文件监听器不跟随或显示符号链接，Web端也不能创建、移动、重命名或删除符号链接。发现符号链接时只记录一次警告，不创建失败任务或反复重试；Docker的Content挂载点本身不受此规则影响。
   - 扫描器和文件监听器只处理最终扩展名不区分大小写为`.zip`的普通文件；`.!qB`、`.part`等下载临时文件因最终扩展名不是`.zip`而自然忽略，不维护下载器专用后缀列表。临时文件重命名为`.zip`后再进入稳定性检查；如果下载器直接写入`.zip`，同样依靠稳定性检查避免提前读取。

#### 文件稳定性与分析一致性

   - 文件事件发生5秒后首次读取文件大小和修改时间，此后每隔5秒检查一次；连续3次结果相同，即稳定至少10秒后，才开始处理ZIP。
   - 等待稳定期间文件状态显示为“等待文件写入完成”。只要大小或修改时间发生变化，就重新计算连续稳定次数，不设置最长等待时间。开始处理前再次读取文件属性；如果与稳定检查结果不同，则返回等待状态。
   - 开始分析时记录路径、设备号、inode、大小和修改时间，分析完成并提交元数据前再次读取。任一属性变化时丢弃本轮分析结果和临时封面缩略图，不覆盖已有有效元数据，并让当前文件重新进入稳定性检查；此情况不计入失败重试次数，也不标记为“处理失败”。分析期间文件被删除时立即隐藏记录并取消相关等待任务。旧版本元数据在新版本分析成功前可继续用于列表展示，图片读取仍以当前文件为准，失败时按既有读取错误规则处理。
   - 使用文件路径、大小和修改时间判断文件变化。文件被外部删除后立即从列表隐藏，遗留元数据和缓存由“清理缓存”功能回收。

#### 失败重试与重新处理

   - 文件首次处理失败后最多自动重试3次，间隔依次为10秒、1分钟和5分钟。等待期间状态显示为“等待重试”，并展示下一次重试时间和最近一次简短错误原因。3次重试均失败后标记为“处理失败”，文件没有变化时不再自动处理。
   - Web手动“重新处理”会开启一轮新的处理，并重新获得最多3次自动重试。文件大小或修改时间发生变化后清除旧失败计数，待文件稳定后开启新一轮处理。
   - Web“重新处理”在`failed`、`no_valid_content`、`ready`和`retry_wait`状态可用，在`waiting_stable`和`analyzing`状态禁用。同一UUID同时最多有一个分析任务，重复点击不重复入队；对`retry_wait`执行时取消等待计时并立即开启新一轮处理，对`ready`执行前要求确认。重新处理前仍执行稳定性检查，并重置最多3次自动重试额度。
   - 新分析失败时，原来有可用元数据则继续保留旧内容并显示“重新处理失败”，否则进入正常失败状态。只有图片清单或文件版本实际变化时才使旧封面缩略图失效；仅文件名解析结果变化不删除封面缩略图。

### 文件身份

   - 每个ZIP文件记录使用服务端生成的随机UUID作为唯一ID，不计算SHA-256。内容相同但路径不同的文件是两个独立文件；文件删除后再次添加时分配新的UUID。
   - 运行期间的移动或重命名通过文件监听事件识别并保留UUID。同一文件系统内，服务重启扫描时仅当旧路径已不存在，且恰好找到一个设备号、inode和文件大小均相同的新路径时，才识别为停机期间发生的移动或重命名并保留UUID。跨文件系统移动按删除旧文件并新增文件处理。
   - 如果旧路径仍存在，则其他新路径均视为独立文件并分配新UUID。如果旧路径不存在但有多个相同设备号、inode和文件大小的候选路径，则不推断移动：旧记录按删除处理，每个候选路径分别分配新UUID。不专门支持硬链接身份同步，不同路径始终作为独立文件管理。
   - 路径和inode不变而大小或修改时间变化时，视为原文件更新，保留UUID和评分并重新分析；同一路径但inode变化时，视为文件替换并分配新UUID。

### ZIP内容处理

   - 首版仅支持未加密的标准ZIP，以及`.jpg`、`.jpeg`、`.png`图片，图片扩展名大小写不敏感。
   - 处理ZIP内相对路径时先将`\\`统一视为`/`。拒绝绝对路径，以及规范化后包含`..`并越过ZIP根目录的路径。忽略目录项和非图片文件；路径中任意一段以`.`开头时排除该条目，任意一段不区分大小写等于`__MACOSX`时同样排除。
   - 剩余图片按相对路径的路径段逐级比较，每段采用不区分大小写的自然排序，使`2`排在`10`之前。同一位置比较时，文件排在同名目录的内容之前；仅大小写不同且自然排序结果相同时，最后按原始UTF-8字节顺序确定稳定次序。
   - 首次分析只读取ZIP中央目录并建立候选图片清单，不验证或解码全部图片；按排序顺序解码第一张可用图片作为封面。没有候选图片，或后来确认全部候选图片均不可用时，标记为“无有效内容”。
   - 损坏、无法读取或需要密码的ZIP仍显示在文件列表中，并标记为“处理失败”。列表显示简短原因，详细异常写入日志，Web文件详情页提供“重新处理”命令。
   - 某页在文件详情或图片浏览中实时读取对应ZIP图片条目时发现无法解压或解码后，将其标记为“不可用”并记录条目名称和失败原因，但保留原序号并计入总页数，不改变其他页码和已有浏览进度。文件详情页在对应位置显示可点击的“页面不可用”占位；图片浏览界面可正常停留在该页，显示全屏空白占位和居中的“页面不可用”，不再次请求原图，也不自动跳过。其余图片仍可阅读，整个ZIP不因此标记为“处理失败”。所有候选页均不可用时标记为“无有效内容”，详情页仍展示占位清单但禁用开始阅读按钮。
   - 只有确定性内容错误才将页面标记为不可用：ZIP条目CRC失败；明确由条目数据损坏导致的解压失败；解压成功但解码器确认不是有效JPEG或PNG；图片违反解压安全限制。Android与服务端网络错误、NAS或ZIP暂时不可访问、服务端资源不足、请求中断、进程重启、未知内部错误或ZIP读取期间变化均不形成不可用结论，按普通读取失败提供“重试”和“返回详情”。
   - 不可用结论保存在元数据中，后续访问直接显示占位。ZIP大小或修改时间变化、文件替换或用户执行单文件“重新处理”时清除旧结论并重新验证。
   - 满足以下任一条件时拒绝处理，并标记为“处理失败：压缩内容异常”：单个条目声明的解压后大小超过512MB；所有条目声明的解压后总大小超过20GB；整体解压后大小超过ZIP压缩大小的200倍且解压后总大小超过1GB；条目存在重叠数据区、异常路径结构或其他无法安全读取的ZIP结构。
   - 解压图片时设置实际输出上限，不能只依赖ZIP头部声明；实际输出超过对应限制时立即中止。详细触发规则和相关条目名称写入日志，Web界面只显示简短原因。

### 图片读取、详情预览图与封面缩略图

  - 新文件通过稳定性检查后立即解析文件名tag、图片清单和封面，并保存相关元数据及封面缩略图。服务端不预先扫描、生成或永久保存详情网格预览图，也不预先解压保存全部原图。
  - “封面缩略图”取排序后的第一张有效图片，在首次扫描分析时自动生成并持久保存在MetaData目录。应用图片方向信息后等比例缩放到最长边不超过720px，不放大小图、不裁切；输出为质量85的JPEG，带透明通道的图片先合成白色背景。第一页原图仍按需从ZIP读取，不在MetaData中重复保存。
  - 详情网格预览图仅在客户端请求时从ZIP条目即时解压、应用图片方向并等比例缩放，不裁切、不放大小图；接口只接受最长边256px、512px和768px三档，输出为质量80的JPEG，带透明通道的图片先合成白色背景。响应ETag由ZIP内容版本、页码和尺寸档位共同确定并设置缓存控制；预览图不写入MetaData或其他服务端持久化目录。相同预览图再次请求时可由客户端或HTTP缓存复用，服务端不维护预览图缓存索引。
  - 文件详情页仅按客户端当前可见范围请求详情网格预览图，不触发后台补全任务；离开详情页后不继续扫描或读取未请求的页面。确定性图片内容错误立即按页面不可用规则处理；NAS暂时不可访问、服务端资源不足或其他可恢复错误按普通图片读取失败返回，不创建持久化生成任务或失败状态。
  - 图片浏览界面继续使用原图接口。原图按请求从ZIP条目边解压边流式返回，不因详情网格已生成预览图而永久解压或生成原图缓存。
  - ZIP更新后旧封面缩略图立即失效，由缓存清理功能回收；新封面缩略图在重新分析时自动生成。有效封面缩略图不按容量自动淘汰。

### 缓存清理与临时文件

  - “清理缓存”只回收已确认删除ZIP对应的元数据、UUID、评分及封面缩略图，ZIP更新后失效的旧封面缩略图，异常中断任务遗留的临时文件，以及数据库中没有归属记录的孤立缓存文件；目录暂时不可访问不视为确认删除。不删除仍存在且未变化的ZIP的元数据、评分和有效封面缩略图。
  - Android和服务端的临时文件超过24小时未修改后视为过期，在应用或服务启动、缓存容量清理及手动“清理缓存”时扫描。当前进程登记为正在下载或生成封面缩略图的临时文件不删除；删除前再次检查修改时间和活动登记。正常任务成功后将临时文件原子替换为正式文件，异常退出遗留文件按24小时规则清理。删除失败只记录警告，不创建高优先级重试任务；临时文件命名必须与正式缓存文件可区分，不能被当作可用图片读取。
  - 执行“清理缓存”前先扫描并显示预计释放空间和涉及的文件数量，经用户确认后执行。清理期间不影响其他有效内容的读取。
  - 同一时间只运行一个“清理缓存”。清理分为扫描和删除阶段，扫描不阻塞正常浏览；删除前再次验证数据仍属于待清理范围。正在通过HTTP返回、正在生成封面缩略图或仍被有效任务引用的缓存本轮跳过且不算失败；其他项目逐个删除，单项失败不阻止后续项。删除顺序为先缓存文件、后对应元数据/UUID/评分记录，且仅在不再被任务使用时删除。完成后显示实际释放空间、成功数量、忙碌跳过数量和失败数量。

### 文件名解析与Tag

  - 首版只从ZIP文件名解析tag，不从父文件夹名称推断，不连接网络源，也不提供手动tag功能。可分类tag仅包括作者、原作和语言；版本、发布组及其他内容统一保存到`unclassified_tags`。作者块同时包含社团名和作者名时，作者tag只使用作者名；只有社团名而没有作者名时，使用社团名作为作者tag。保存原始文件名、解析结果、置信度和解析警告；解析失败时使用完整basename作为显示标题，不影响文件导入和阅读。解析规则见`/docs/tags-rules.md`。
  - 作者、原作和语言tag均允许多个值，每个值作为独立tag保存；只按tag规则中明确支持的分隔符拆分，无法可靠拆分时保留为单个tag。列表摘要按作者、原作、语言的固定顺序显示，同类内部保持文件名中的出现顺序，值之间使用` · `分隔。对Unicode NFC规范化后且忽略大小写相同的值去重，显示时保留首次出现的原始文本。筛选多个tag时仍使用“全部匹配”或“匹配任一”规则，不因tag类别不同而改变。
  - Web重命名或外部重命名导致文件名变化后自动重新解析，并完整替换旧的文件名解析结果；仅移动目录且文件名不变时不重新解析。元数据记录解析器版本，解析器升级后执行“重新扫描文件”时重新解析版本过旧的记录。

### 列表、搜索与筛选接口

  - 所有文件列表、搜索和筛选接口必须服务端分页，默认每页50条，最大每页200条。
   - 普通列表和搜索不创建数据快照，每次查询当前最新状态；单次响应内部结果一致且排序稳定，并返回结果版本标识。相邻分页版本变化时，Android按UUID去重并提示列表已更新，可继续浏览，但需下拉刷新才能重新加载可能插入到已浏览区间的文件；Web保留筛选和排序条件并提示刷新，刷新后回到第一页。文件删除后立即从客户端当前已显示列表移除。不承诺文件持续变化期间跨页结果具有事务快照一致性，跨分页批量操作仍使用独立的UUID选择快照。
   - 默认按文件名自然升序排列。文件夹始终排在ZIP之前，不受升序或降序影响；文件夹之间始终按名称自然升序排列，不递归计算目录大小，也不按评分或目录修改时间排序。
   - ZIP支持按文件名、文件大小、修改时间和评分排序，并支持升序和降序。主排序值相同时按文件名自然升序排列，名称仍相同时按UUID升序，保证分页顺序稳定。评分升序时0星排在有评分文件之前，降序时0星排在最后。
   - 文件名搜索、tag筛选和评分筛选可以组合，多种条件之间采用“并且”关系。多个tag可选择“全部匹配”或“匹配任一”。
   - tag选择器按作者、原作和语言三类分组。打开时每类先显示所选目录范围内使用次数最多的20个tag；输入关键字后由服务端按Unicode NFC规范化后的不区分大小写字面子串匹配，每类最多返回50个结果。每项显示tag值及当前目录范围内匹配的ZIP数量，数量遵循“仅当前文件夹/包含子文件夹”范围但不受当前已选tag影响。已选tag始终显示；不同类别中的同名值不合并。未分类tag首版只在详情页展示，不参与筛选。
   - 评分筛选采用包含边界的整数闭区间，最小值和最大值均为`0...5`且最小值不得大于最大值。默认`0...5`等同于未启用；`0...0`只显示未评分，`0...3`表示3星及以下且包含未评分，`4...5`表示4星及以上。Android和Web使用相同参数，清除筛选恢复为`0...5`。
   - 文件名搜索对象包括所选范围内的ZIP文件名和文件夹名称，ZIP匹配包含`.zip`扩展名。输入去除首尾空白，空字符串等同于未启用搜索；查询和名称均执行Unicode NFC规范化后进行不区分大小写的字面子串匹配，不支持正则表达式、通配符或模糊拼写纠正，也不合并查询中的连续内部空格。搜索结果继续与当前排序、tag、评分和目录范围条件组合。
   - 搜索和筛选范围可选择“仅当前文件夹”或“包含子文件夹”，默认仅当前文件夹。
   - 未启用搜索或筛选时，显示当前目录的直接子文件夹和ZIP。仅启用文件名搜索时，显示名称匹配的文件夹和ZIP；启用tag或评分筛选后只显示符合条件的ZIP，不显示文件夹行。
   - 范围为“仅当前文件夹”时只匹配当前目录直接包含的ZIP；范围为“包含子文件夹”时以扁平结果显示所有匹配ZIP，每行额外显示相对于当前目录的路径。点击结果直接进入详情，返回时恢复原筛选条件和滚动或分页位置；清除tag和评分筛选后恢复文件夹导航。

### 存储卷与备份

  - docker挂载3个文件夹：
   - MetaData文件夹：用于储存各ZIP文件的元数据、UUID和评分
   - Config文件夹：储存服务端配置、管理员凭据、APIkey和日志
   - Content文件夹：存放和管理同人志zip文件的文件夹
  - MetaData和Config必须使用持久化卷，Docker更新或重建不得删除。首版不提供应用内备份和恢复功能，只保证冷备份流程：停止Dokura容器后，使用同一备份批次完整备份MetaData和Config，完成后再启动；恢复时同样先停止容器，用同一批次完整替换两个目录后再启动。不支持混用不同时间点的MetaData和Config。只有NAS能保证两个卷的一致性快照时，才支持运行中创建文件级快照。封面缩略图缓存可以不备份，恢复后可在扫描分析时重新生成。
  - Content不属于Dokura配置备份，用户应通过NAS策略独立备份原始ZIP。恢复MetaData和Config后，启动扫描根据当前Content校正文件状态，备份后新增或替换的ZIP按现有身份规则处理。

### Android API鉴权

  - 服务端和安卓客户端之间采用APIkey鉴权。安卓客户端不能调用删除、移动、重命名和后台处理控制等管理接口。
  - 首次启动自动生成一个高强度随机APIkey，不使用固定默认值。Web设置页平时只显示末尾4个字符，不允许再次查看完整旧key；完整key只在生成成功后的确认窗口显示一次，关闭后只能再次生成。用户关闭前需确认已经记录。
  - 重新生成APIkey前要求输入当前管理员密码并二次确认，明确所有Android客户端均需重新配置。新key生成成功后旧key立即失效；使用旧key且已经开始处理的请求允许完成，后续请求拒绝。APIkey不得写入普通日志、下载日志、URL或错误信息，Android使用系统安全存储保存，不随普通界面状态导出。

### 后台任务调度

  - 后台服务运行期间始终按优先级处理文件监听、扫描、ZIP分析和失败重试；封面缩略图生成属于ZIP分析的一部分，详情网格预览图属于前台请求，不建立内容图片生成队列。后台处理不提供全局或单任务的暂停、继续和取消功能，停止全部后台处理只能通过Docker或NAS管理界面停止容器。
  - 后台任务意图、必要状态和重试额度持久化到MetaData。服务启动时将上次正在执行的ZIP分析任务恢复为等待队列，不计为失败；`retry_wait`保留已使用次数和下次执行时间，时间已过则启动后按优先级执行。启动扫描与恢复任务按UUID和任务类型去重，未原子提交的临时封面缩略图不算成功并按临时文件规则处理。
  - Android和Web的列表、详情、评分、封面缩略图、详情网格预览图及原图读取属于前台请求，不进入后台队列。后台优先级从高到低为：文件删除/移动/重命名事件的状态同步；新增或已修改ZIP的稳定性检查与分析；用户触发的单文件“重新处理”；到期自动重试及“重试所有失败”加入的任务；“重新扫描文件”的目录遍历及解析器旧版本重解析。
  - 已开始的单个ZIP分析不中断，完成当前单元后选择最高优先级任务；同一优先级按入队时间处理，相同UUID和任务类型去重。低优先级任务连续等待超过10分钟后提升一级，但不能提升到文件事件状态同步之上。Web显示各优先级等待数量，不提供手动调整优先级的功能。
  - “重试所有失败”只处理文件分析状态为`failed`的ZIP，以及保留旧元数据但最近一次重新处理失败的ZIP；不处理`no_valid_content`或已标记不可用的单张图片。执行前显示ZIP数量并要求确认；每个文件重新执行稳定性检查，开启新一轮处理并重置最多3次自动重试。已不存在的文件跳过并隐藏，已在等待或分析队列中的UUID不重复加入。命令提交任务后立即返回，由后台按队列处理，Web显示已加入、跳过和提交失败数量。

### 日志

  - 生成日志文件并存放在Config卷中。总共最多保留10个日志文件，包括1个当前活动日志和最多9个轮转日志。活动日志达到20MB时立即轮转并删除最旧文件使总数不超过10；服务重启继续写入未满上限的活动日志。文件名包含UTC创建时间和递增序号，避免时钟变化导致重名。日志不得记录密码、APIkey等鉴权信息；正常的单张图片请求不记录信息级日志。Config卷空间不足导致日志无法写入时，核心浏览功能继续可用，Web状态显示日志写入异常。

### 路径安全

  - 每次读取ZIP以及删除、移动或重命名前，都解析目标的真实路径并再次验证其位于Content根目录内；校验失败时拒绝操作并记录警告。

## Web界面

### 登录、密码与会话

  - web界面采用账号密码登录，默认账号admin，密码admin。首次登录后醒目提示用户修改默认密码，但不强制修改。
   - 密码仅保存强哈希，不保存明文。Web登录会话连续24小时无操作后过期，最长有效期为7天；用户操作只延长空闲期限，不延长7天的绝对期限。主动退出后服务端立即撤销当前会话，修改管理员密码后撤销全部现有Web会话并要求重新登录。
   - 新密码长度为8至128个Unicode字符，不强制大小写、数字或特殊字符组合，允许密码短语；不得等于默认密码`admin`，也不得与当前密码相同。Web修改密码时要求输入当前密码，并两次输入一致的新密码。密码首尾空白属于密码本身，不自动裁剪，界面需明确提示。
   - 连续输错当前密码使用现有登录失败限流规则。Docker密码重置命令不要求旧密码，但新密码遵循相同长度和默认密码限制。密码修改成功后不自动重新生成Android APIkey。
   - 同一来源IP或同一账号在15分钟内累计登录失败5次后，禁止继续登录15分钟，任一维度触发即限流；登录成功后清除该账号和来源IP对应的失败计数。限流界面显示剩余等待时间，但不透露账号是否存在。会话记录持久化，服务重启不使尚未过期的会话自动失效。
   - 不提供邮箱找回密码，通过Docker命令重置管理员密码。

### 功能范围与跨端边界

  - Web提供文件列表、文件详情、图片浏览、设置和后台日志界面。文件列表与Android共同支持目录浏览、文件名搜索、作者/原作/语言tag筛选、评分区间、范围选择及排序；文件详情共同显示既定元数据、按可见范围实时加载的内容预览图、评分和进入图片浏览。两端共用相同的服务端数据、状态码、排序和筛选语义，布局及交互分别按各自章节实现。
  - Web不保存浏览进度，因此不显示“从X页开始浏览”，统一提供“从第1页浏览”，也可点击详情页中已加载的指定预览图进入对应页。Web不提供最近阅读、Android本地图片缓存、预读取行为设置、阅读方向、保持屏幕常亮或服务端连接配置。
  - Android不提供文件管理、后台任务控制、日志、服务端缓存管理、APIkey生成或管理员功能。Web图片浏览只采用本章节的桌面规则，不继承Android的缩放和触摸手势。

### URL与导航状态

  - 当前目录、页码、每页数量、排序、搜索词、搜索范围、tag筛选及评分区间写入URL查询参数。文件详情和图片浏览使用包含UUID的独立URL，图片浏览URL同时包含当前页码但不视为浏览进度。浏览器前进、后退和刷新均恢复URL状态；无效参数回退到默认值并从URL移除。已删除或无权限的UUID显示明确错误并提供返回列表按钮。
  - 跨分页选择快照、删除确认框和登录会话不写入URL，也不能通过复制链接共享；URL不得包含APIkey、会话标识或其他秘密。

### 文件详情与图片浏览

  - Web图片浏览界面采用独立的桌面交互：图片始终等比例适应浏览器可用区域，完整显示且不裁切，不提供缩放、实际像素切换、拖动平移、双击或触摸滑动。图片左右两侧固定显示上一页和下一页箭头按钮，到达边界时对应按钮禁用。
  - Web内容预览图网格按可用宽度响应式排列，单张图片宽度不小于160px且不大于240px，不使用Android的列数设置。网格只按当前可见范围请求详情网格预览图；浏览器根据CSS尺寸和设备像素比，从256px、512px和768px三档中选择不小于展示尺寸的最小档位，超过768px时使用768px档。未加载时显示固定尺寸占位，不预先请求全部页面；进入图片浏览界面后才请求原图。
  - Web支持键盘`←`、`→`翻页，`Home`进入第1页，`End`进入最后一页，`Esc`返回详情页。底部固定显示当前页/总页数和页码滑动条；翻页时显示加载占位并按需读取原图。阅读方向固定从左向右，不记录进度；浏览器窗口尺寸变化时重新计算适应尺寸并保留当前页。

### 文件管理

#### 跨分页选择快照

   - Web使用常规分页控件。支持跨分页选择完整筛选结果，而不是只选择当前已加载页。用户选择全部结果时，服务端创建仅在当前登录会话内有效的选择快照，保存当时匹配的ZIP UUID集合；每个会话最多保留一个，新快照自动释放旧快照。快照连续30分钟没有选择、取消选择或批量操作交互后过期，退出登录、取消选择或操作完成后立即释放。后来新增或变得符合筛选条件的文件不自动加入快照。
   - 页面刷新可通过会话恢复未过期快照，浏览器关闭不立即释放但仍按30分钟过期。快照过期后禁止批量操作并提示重新选择；服务重启后全部快照失效且不持久化。打开删除确认框时仍按既有规则重新校验集合、数量和总大小。

#### 支持的文件与文件夹操作

   - 文件支持删除、移动和重命名；支持多选ZIP后批量移动。首版不支持批量重命名、上传ZIP、解压ZIP或创建压缩包。
   - 文件夹支持新建、重命名和移动。新建和重命名文件夹使用下述通用名称校验规则，但不附加`.zip`扩展名。禁止将文件夹移入自身或其子目录，不提供递归删除。只允许删除文件系统实际为空的文件夹；存在任何文件、子目录、隐藏项、非ZIP文件或符号链接时均拒绝删除，即使这些内容未显示在Dokura列表中。删除前立即重新检查，系统删除仍因竞态失败时保留目录、显示简短错误且不自动重试；Web可提示存在未管理内容，但不展示或操作这些内容。
   - Web成功移动或重命名文件夹时，在同一管理操作中更新所有后代ZIP和子文件夹的相对路径；后代ZIP保留UUID、评分、封面缩略图关联和Android本地进度，文件名未变化时不重新解析tag。执行前检查整个目标子树的名称冲突和路径长度，任一不满足则不执行；操作期间禁止对子树发起其他移动、重命名或删除。
   - 外部工具在运行期间移动同文件系统文件夹时，优先使用目录移动事件整体更新；监听器只收到分散事件时，通过设备号和inode对账并尽量保留后代UUID。外部跨文件系统文件夹移动仍按每个ZIP删除后新增处理。

#### 名称、路径与移动校验

   - 移动只能选择Content内已有文件夹作为目标。重命名输入只能是单个名称，禁止`/`、`\\`、NUL、其他控制字符及`< > : " | ? *`；去除首尾空白后不得为空，不得以空格或句点结尾，并禁止`.`、`..`以及不区分大小写的Windows保留名`CON`、`PRN`、`AUX`、`NUL`、`COM1`至`COM9`、`LPT1`至`LPT9`。单个名称按UTF-8编码后不得超过255字节。
   - Web移动要求源目录和目标目录的设备号相同；目标虽位于Content内但属于不同文件系统时预检失败，并提示“首版不支持跨文件系统移动”。ZIP和文件夹均适用此限制，首版不实现复制后删除式移动。同文件系统Web移动使用原子重命名并保留ZIP的UUID、评分和Android本地进度；外部工具执行跨文件系统移动时仍按删除旧文件和新增文件处理。
   - ZIP重命名界面只编辑不含扩展名的basename，由系统保留原有`.zip`扩展名。目标目录内按不区分大小写检查同名冲突，但允许只修改名称大小写。校验失败时不自动替换字符，界面明确显示原因并由用户修改。
   - 单个路径段按UTF-8编码后最多255字节，从Content根目录开始的完整相对路径按UTF-8编码后最多4095字节，ZIP的`.zip`扩展名计入两项长度。创建、重命名或移动前检查操作后所有受影响路径，不自动截断。外部创建的超长路径无法安全访问时记录警告且不导入；实际文件系统限制更严格时以其错误为准并显示失败原因。

#### 批量操作、并发与删除

   - 目标路径存在同名文件或文件夹时对应项目操作失败，绝不覆盖。批量移动对每个项目独立校验源文件UUID和路径、目标目录、名称冲突及基本权限；冲突或执行失败的项目跳过，其余项目继续，不回滚成功项。完成后分别列出成功项、失败项和简短原因，详细错误写入日志，并对涉及的源目录和目标目录执行增量扫描，以实际文件系统状态校正数据库。成功移动或重命名后保留UUID和评分。
   - 每个UUID的ZIP分析、单张原图读取和Web管理操作互斥。删除、移动或重命名最多等待当前分析或单张原图读取5秒；获得操作权后执行，超时则该项目返回“文件正在处理，请稍后重试”，不自动排队。操作成功后，尚未开始的旧路径任务按UUID更新路径，删除成功则移除相关等待任务；操作失败不影响原后台任务。外部文件操作仍由处理前后属性校验和增量扫描校正。列表允许选择正在处理的文件，确认窗口提示这些项目可能因忙碌失败。
   - 删除为永久删除，不提供回收站。批量删除确认窗口基于选择快照中当前仍存在且仍符合原筛选条件的文件，重新计算文件数量和总大小；已删除或不再符合条件的文件从待删除集合排除。用户确认时服务端再次校验快照，如果数量或总大小与确认窗口显示值不同，则不执行删除，刷新统计并要求再次确认；实际删除只作用于最终确认的UUID集合。
   - 批量删除执行前检查所有目标仍位于Content目录内、仍对应原UUID，并检查基本删除权限；通过预检后逐个删除，单个失败不阻止继续处理其他文件。完成后分别显示成功数量、失败数量和失败项的简短原因，详细错误写入日志。成功项立即从列表隐藏，失败项继续保留；不提供事务回滚。再次删除失败项时仍需重新确认。
   - 必须支持“筛选出3星及以下文件（含0星），跨分页全选并批量删除”的流程。

### 设置与后台任务

## 技术栈

 - 部署：docker
 - 代码：python
 - python包和虚拟环境管理：uv
 - web服务后端：FastAPI
 - web服务前端框架：Vue 3 + Naive UI
 - 元数据储存：SQLite，通过同步SQLAlchemy 2.x访问并使用Alembic管理数据库结构迁移。数据库文件存放在MetaData卷中，启用WAL、外键约束、`synchronous=FULL`和5秒`busy_timeout`；MetaData卷使用支持可靠文件锁的本地或块存储，不将数据库文件直接放在不可靠的网络共享文件系统中。首版只运行一个Dokura服务进程，并在应用内保证同一时间只有一个写事务。
 - Docker镜像内Python实际链接的SQLite运行时版本必须为3.51.3或更高版本。镜像构建测试和服务启动时均检查`sqlite3.sqlite_version`；版本不足时拒绝启动并记录明确原因，不能仅以Python包或系统命令显示的版本代替运行时检查。
 - 数据库写事务必须保持短小，事务打开期间不得执行ZIP读取、图片解码、网络请求或其他可能长时间阻塞的文件系统操作。扫描和批量元数据更新每个事务最多提交200个ZIP记录，必须保持原子性的单个管理操作除外；批次之间允许前台读请求执行。读取使用独立短事务，响应流式图片期间不得保持数据库事务或连接。
 - WAL使用默认1000页自动checkpoint阈值；大量扫描或批量操作完成且没有活动写事务时执行一次被动checkpoint。服务应监控WAL文件大小并记录异常增长；不得通过长期读事务阻止checkpoint，也不得在正常前台请求中执行会阻塞读写的强制checkpoint。
 - ZIP、原图和封面缩略图不写入SQLite，数据库只保存其元数据、UUID、状态、评分、任务和缓存索引。数据库迁移在服务启动且对外提供管理功能前执行；自然排序键、规范化搜索字段和常用筛选字段预计算并建立索引，以满足100,000条元数据的性能目标。
