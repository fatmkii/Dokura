# 阶段 7 验收记录

## 范围

- Android 全屏阅读器、左右阅读方向、边缘点击、适屏/双指缩放、放大后平移、正序页码滑动条与旋转恢复。
- 适屏采样解码、放大后的可见区域解码，以及普通/低内存设备位图预算和内存压力释放。
- 当前页保留槽、前 3 后 10 页预读取、窗口取消、JPEG/PNG 校验、临时文件和原子缓存替换。
- 1/5/10/20GB 类别优先 LRU、保护集合、Room 索引、30 秒/100 次访问合并、启动对账和安全清空。
- 页面稳定显示 500ms 后保存进度，首次显示、每分钟及退出时更新最近阅读。

## 一键验收

```bash
scripts/acceptance/stage-07.sh
```

脚本运行 Android JVM 测试、lint 和 APK 构建，再通过 Windows SDK 的 `adb.exe` 在 `emulator-5554` 安装并执行仪器测试。可使用 `ADB_EXE` 和 `ANDROID_DEVICE` 覆盖默认路径与设备。

测试只使用 JVM 策略数据、模拟器内的内存 Room 数据库和独立临时缓存目录；不会读取、修改或删除仓库 `Content/`。

## 2026-07-17 验收结果

- 环境：WSL2 Ubuntu 24.04.1、JDK 17、Gradle 9.5、Android SDK 36、Windows `emulator-5554`（Android 36）。
- JVM 测试覆盖预读取页码顺序、阅读方向手势、放大后拖动不翻页、当前页保留槽、普通/低内存预算、采样率和类别淘汰顺序。
- 模拟器仪器测试覆盖左右方向滑动、放大后拖动不翻页、放大后的边缘点击、缓存类别淘汰和保护、精确容量、清空缓存保留阅读进度、异常退出后的索引对账、网络失败不遗留临时文件，以及阶段 6 的导航、旋转、主题和字体缩放回归。
- `lintDebug`、`assembleDebug`、`assembleDebugAndroidTest` 通过。

阶段 7 不建立完整离线书库；磁盘缓存只用于在线阅读加速，符合首版范围边界。
