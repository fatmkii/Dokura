# 阶段 4 验收报告

## 范围

- Web 登录、纵向目录列表、面包屑、分页及完整 URL 查询状态。
- 300ms 搜索防抖、目录范围、tag、评分区间、排序及评分乐观更新/失败回滚。
- 文件详情分组元数据、按可见范围加载的响应式预览网格和页面不可用状态。
- 独立全屏阅读器、原图按需读取、键盘翻页、边界禁用及窗口尺寸变化。
- 跟随系统/亮色/暗色主题，以及窄屏、桌面宽屏和 200% 字体缩放布局。

## 一键命令

```bash
scripts/acceptance/stage-04.sh
```

脚本使用临时 Content、MetaData 和 Config 目录构建并启动生产 Docker。Playwright 的确定性目录、详情与图片响应在浏览器层拦截；不会读取、修改或删除仓库 `Content/` 中的样例。

## 自动断言

- 登录成功进入内容库，默认密码提示可见；详情与阅读器 URL 在生产 FastAPI 静态托管下可直接刷新。
- 目录、页码、每页数量、搜索、范围、tag、评分和排序经刷新及浏览器返回后恢复，无效值由状态解析器回退。
- 连续搜索输入只在 300ms 后提交最后值；筛选与排序即时写入 URL。
- 评分先更新界面，服务端失败后恢复原值并显示中文提示。
- 详情初次只请求可见范围附近的预览，不请求全部 40 页；进入阅读器后才发起原图请求。
- 阅读器覆盖方向键、Home、End、Esc、首尾按钮禁用和视口变化保留页码。
- 暗色主题刷新后保留；360px 窄屏、1600px 宽屏与 200% 根字号下关键搜索控件可见且页面无横向溢出。

## 已知边界

- 阶段 4 只实现浏览与阅读闭环；设置、密码修改、文件管理、任务和日志 UI 属于阶段 5。
- Playwright 使用生产 Docker 中的真实前端构建，但通过浏览器网络拦截构造可重复的数据与图片；真实 201 个 ZIP 的只读全量回归仍由阶段 9 统一执行。
