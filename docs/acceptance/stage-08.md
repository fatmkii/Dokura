# 阶段 8 验收记录

## 范围

- 固定补丁版本和镜像摘要的生产构建、UID/GID 10001 非特权运行、只读根文件系统、健康检查及启动迁移。
- MetaData/Config 同批次冷备份、归档校验后完整恢复、容器重建、Content 恢复后身份对账和缺失封面再生成。
- 路径与 ZIP 安全、鉴权和会话撤销、限流、日志脱敏、图片队列背压及资源释放集中回归。
- 用户首次部署、首次登录、Android APIkey、冷备份恢复、升级和回退文档。

## 一键验收

```bash
scripts/acceptance/stage-08.sh
```

脚本使用 `/tmp/dokura-stage-08.*` 下的一次性 Content、MetaData 和 Config，绝不读写仓库中的真实内容样例。它会构建 `dokura:stage-08`，启动临时 Compose 项目并占用端口 `18008`；可通过 `DOKURA_STAGE8_PORT` 覆盖端口。

验收依次覆盖从阶段 2 迁移版本升级到 head、服务端完整回归、容器安全属性、Content 权限故障、同批次冷备份恢复、强制重建和三轮重启。归档校验和、凭据一致性、容器健康状态、运行 UID/GID 及 `/tmp` 残留均由脚本断言。

## 2026-07-17 验收结果

- 环境：WSL2 Ubuntu 24.04.1、Docker 29.4.3、Docker Compose 5.1.4、uv 0.11.16、CPython 3.14.5。
- 前一迁移版本 `9bc218db86e1` 升级到 `43d1a9b0c7ef` 后 Alembic 无漂移。
- 服务端 94 项测试通过，覆盖扫描中健康响应、Content 不可访问保护、会话/APIkey 撤销、路径/ZIP/日志/队列安全和缺失封面再生成。
- `dokura:stage-08` 构建通过；运行用户为 `10001:10001`，根文件系统只读，Content 权限故障期间健康检查保持通过。
- 同批次冷备份 SHA-256 校验、完整恢复、凭据一致性和强制容器重建通过。
- 三轮重启后 uvicorn 文件描述符未持续增加、线程数一致、活动任务为零且 `/tmp` 无文件残留。

数据集全部位于脚本生成的临时卷；仓库 `content/` 中的真实 ZIP 未读取或修改。阶段 9 的 100,000 条规模性能、60 分钟资源趋势和 Android 长时阅读不属于本阶段验收范围。
