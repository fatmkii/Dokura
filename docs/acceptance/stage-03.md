# 阶段 3 验收报告

## 范围

- 目录与 ZIP 服务端分页、稳定排序、NFC/casefold 字面搜索、目录范围、tag 与评分组合筛选。
- FTS5 trigram 候选、tag 关联、父目录/评分/大小/修改时间组合索引和目录元数据。
- 文件详情、幂等评分、封面 Range、256/512/768 即时 JPEG 预览及 ZIP 原图流式响应。
- 图片用途优先级、3 个读取槽位（当前页保留 1 个）、2 个解码槽、64/32 有界队列、并发预览合并和 128MB 编码结果 LRU。
- Web 登录限流、24 小时空闲/7 天绝对会话期限、Origin 校验、密码修改与 Docker 重置、Android APIkey 生成/轮换和权限隔离。

## 一键命令

```bash
scripts/acceptance/stage-03.sh
```

脚本使用 pytest、`mktemp` 和临时 SQLite 数据库；图片与 ZIP 均在测试临时目录生成，不读取、修改或删除仓库的 `Content/` 样例。

## 自动断言

- OpenAPI 声明 Web HttpOnly Cookie 与 Android Bearer 两种安全方案；管理接口只接受 Web 会话，Android 凭据只可调用浏览与评分接口。
- 同一来源 IP 或账号 15 分钟内五次失败触发 15 分钟限流并返回 `Retry-After`；密码修改/重置撤销会话，APIkey 轮换立即使旧 key 失效。
- 目录永远排在 ZIP 之前；自然排序、分页、tag 全部/任一、评分闭区间和评分幂等更新均逐字段断言。
- 1–2 字符扫描与三字符以上 trigram 候选都经过 NFC/casefold 精确子串后过滤，组合条件由批量 tag 查询完成，无逐行 N+1。
- 封面支持 ETag、304 和单段 Range；预览图最长边及 JPEG 输出、原图字节流、内容版本 ETag 失效和确定性页面不可用持久化均通过。
- 当前页可使用保留读取槽；预览/预读取不能占满第三槽。单客户端 32、全局 64 队列上限、取消移除、`Retry-After`、并发预览合并和 LRU 字节上限均有自动测试。
- 生成 100,000 条、100 个目录和代表性 tag/评分分布的元数据，临时数据库约 157MB；查询计划分别命中 `ix_files_parent_name`、`ix_files_parent_rating`、`ix_file_tags_tag_file` 和 FTS5 虚拟表索引。
- 当前验收环境完整服务端回归为 71 项测试（后续补充测试后以脚本实时输出为准），OpenAPI 重复生成无漂移。

## 已知边界

- 阶段 3 交付服务端 API；Web 页面和 Android 客户端分别在阶段 4 与阶段 6 接入。
- 详情预览只缓存编码后的内存结果且不落盘；原图仍按请求从 ZIP 流式读取，符合首版服务端缓存边界。
- 阶段 9 才执行参考硬件上的 P95、60 分钟资源稳定性和混合前后台负载指标；本阶段只验证查询计划、队列边界和功能语义。
