# 阶段 9 验收记录

## 范围

- 固定 100,000 条代表性元数据及匹配扫描文件树，另以真实 ZIP 验证 ZIP 分析、原图读取和 768px 预览生成。
- 容器 HTTP 列表、搜索、筛选、排序、封面、预览、原图和评分性能矩阵。
- 60 分钟“完整扫描 + 前台浏览”混合负载，以及 RSS、文件描述符和线程趋势。
- Android 60 分钟连续下载、采样解码和热缓存翻页，接近 20GB 的稀疏缓存边界、主动清空和断网恢复。
- 阶段 0～8 全量回归，以及 Docker 镜像、Web 静态资源和 Android APK 两次独立构建一致性。

## 一键发布验收

```bash
DOKURA_ANDROID_BASE_URL=http://<WSL局域网地址>:18009 \
DOKURA_RELEASE_STORAGE='MetaData=本地SSD；Content=顺序读取xxxMB/s NAS' \
DOKURA_RELEASE_NETWORK='服务端与客户端千兆局域网' \
scripts/acceptance/release.sh
```

正式配置固定使用 100,000 条数据、服务端 60 分钟稳定性测试、Android 60 分钟阅读和 20GB 缓存边界，任何指标失败都会令脚本失败。结果 JSON 会写入脚本最后打印的临时报告目录，包含硬件、网络、存储、数据集、Git 版本及冷热缓存条件；可用 `DOKURA_RELEASE_REPORT_DIR` 指定持久化目录。

脚本只读访问仓库 `content/`。扫描、数据库、图片缓存和 Android 缓存边界测试均使用 `/tmp/dokura-stage-09.*` 或模拟器应用的独立临时缓存，清理不会修改真实 ZIP。

## 脚本自检

开发过程中可显式设置 `DOKURA_RELEASE_PROFILE=smoke` 缩短数据集和持续时间。smoke 会明确跳过阶段 0～8 和可复现构建，并继续执行后续流程以收集指标失败报告；输出会标记“不构成发布验收”。正式配置不会读取这些缩短参数，也不会容忍指标失败，因此不能静默降低 PRD 指标。

- 阶段 0～8 全量回归通过；201 个真实 ZIP 全部只读处理且目录零修改。另按大小等距选取 9 个真实 ZIP 执行分析、原图读取和 768px 预览基准，全部 SHA-256 不变。
- 固定数据集包含 100,000 条元数据、100 个目录和 30 个 tag。首次完整扫描耗时 290.780 秒；60 分钟混合负载中的 12 次完整扫描耗时 293.307～297.053 秒，均低于 600 秒门槛。

- Web 静态资源、Android debug APK 和生产 Docker OCI 镜像均完成两次独立构建并逐字节一致；两次 OCI 清单均为 `sha256:c3d76eabc67f63b263ea6a76a589465320d8b8553306206bf88446ec41133bc8`。
