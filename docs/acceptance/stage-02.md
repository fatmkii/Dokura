# 阶段 2 验收报告

## 范围

- 启动扫描、手动完整扫描、递归文件监听与一秒事件合并。
- 5 秒集中稳定性调度、连续三次稳定检查和 10 秒/1 分钟/5 分钟重试。
- UUID、设备号、inode、路径、大小和修改时间对应的移动、重命名、更新与替换规则。
- 持久化任务恢复、活动任务去重、优先级等待老化、单分析调度及前台压力让路。
- 不可访问目录保护、扫描状态、确认删除和服务端缓存清理预检/执行。
- 流式目录遍历、每批最多 200 条和 SQL 查询计数回归。

## 一键命令

```bash
scripts/acceptance/stage-02.sh
```

脚本只在 pytest 和 `mktemp` 创建的临时目录内执行文件新增、连续写入、修改、删除、移动、重命名、替换、权限异常、重试与缓存删除；不会写入或删除仓库的 `content/` 样例。

## 自动断言

- 符号链接、非 ZIP 和临时扩展名不创建文件记录或分析任务。
- 同一文件的重复扫描只保留一个活动分析任务；调度器只运行一个分析阶段。
- 连续写入会重置稳定计数；中断的 `analyzing` 任务重启后恢复且不重置已用重试额度。
- 同盘移动/重命名和原位更新保留 UUID 与评分；同路径 inode 替换产生新 UUID 且不继承评分。
- 根目录或子目录不可访问时记录暂时不可访问，不确认删除元数据、UUID、评分或封面。
- 扫描运行期间的重复请求折叠为恰好一次后续扫描。
- 401 个 ZIP 的扫描按 200 条分批，文件和任务查询数量保持批次数量级。
- 缓存清理只执行已预检并确认、执行时仍然无效的项目，顺序为先缓存文件后元数据。

## 已知边界

- 阶段 2 暴露管理扫描、任务状态和缓存清理的服务端接口；Web 管理界面及管理鉴权分别在阶段 5 和阶段 3 完成。
- 文件监听使用 `watchfiles` 的递归监听与 1000ms 合并窗口；监听暂时不可用时服务保持健康并通过保护性扫描恢复。
