# Dokura客户端需求

本文件应与[共通需求](./PRD.md)一并阅读。后台服务和Web界面的需求见[服务端需求](./PRD-server.md)。

## Android应用
  - 客户端仅需求安卓apk版本，不需要考虑apple版本。
  - 特殊功能：
   - 缓存功能：缓存封面缩略图、内容缩略图和已读取的原图，默认上限为5GB。设置提供1GB、5GB、10GB、20GB四档。缓存用量按应用缓存目录内上述三类文件及下载临时文件的实际磁盘字节总和计算，统计误差不超过1MB；临时文件先计入容量，下载成功后原子替换为正式缓存文件。
   - 每次缓存写入完成后检查上限。超限时先删除未完成和过期临时文件，再依次淘汰原图、内容缩略图和封面，每个类别内部按最后访问时间从旧到新淘汰，只有前一类别没有可淘汰项时才进入下一类别。当前显示页和预读取队列中的文件临时受保护，离开阅读界面后解除保护；如果全部可淘汰项均受保护，则停止新的预读取，当前页仍可流式加载和显示但不强制写入缓存。调低缓存上限后立即执行相同淘汰流程，完成后刷新实际用量。
   - 清空缓存只删除封面、内容缩略图、原图和临时文件，不删除服务端连接配置、APIkey、评分、浏览进度、最近阅读、主题或其他设置。执行前显示缓存用量并要求确认，同一时间只允许一个清空操作。
   - 用户确认后立即移除尚未开始的预读取任务；正在进行的预读取和图片下载可完成网络响应，但不再写入磁盘缓存。当前屏幕已解码的图片保留在内存中直到离开页面。清空期间新打开图片可流式显示但不写入缓存，结束后恢复正常缓存。单文件删除失败时跳过并继续，完成后显示实际释放空间和失败数量。
   - 预读取功能：进入图片浏览界面后，预读取窗口始终以当前成功显示页为中心，包含之前3页和之后10页，跳过越界、已知不可用及已有有效缓存的页面。当前页请求优先级最高；最多同时执行3个原图网络请求，并至少保留1个并发槽位给当前页。设备可用空间不足时停止预读取，但仍允许按需加载当前页。
   - 当前页变化后，移除尚未开始且已移出新窗口的预读取任务；已经开始的窗口外请求不强制中断，完成后仍可缓存。离开图片浏览界面后移除所有尚未开始的预读取任务，正在传输的请求可完成并缓存。阅读方向只影响手势，不反转“前3页、后10页”的页码含义。
   - 评分功能：用户对当前ZIP设置`0、1、2、3、4、5`中的整数评分，不支持半星；0表示未评分，也是默认值，可用于清除已有评分。评分在服务端仅保存一个值，最后成功写入的请求生效；接口返回最终分值和服务端更新时间，Android和Web可先乐观更新，失败时恢复原值并提示保存失败。文件内容更新且保留UUID时保留评分；文件替换、删除后重新添加或产生新UUID时不继承评分。
   - 浏览进度：按文件UUID在本地记录当前浏览页码和最后阅读时间，下次直接跳转，不跨设备同步。某页成为阅读器当前页，且原图、可用内容缩略图或“页面不可用”占位成功显示并稳定500毫秒后更新页码；预读取完成、尚未形成不可用结论的图片加载失败，以及尚未成功显示的“从X页开始浏览”均不更新进度。连续快速翻页采用上述防抖，退出阅读器或应用进入后台时立即写入最后一个已成功显示的页面；本地写入失败不阻止阅读，下次成功翻页时重试。文件内容更新后若记录页码超过新的总页数，则调整到最后一页。
   - 最近阅读：在文件列表页提供最近阅读视图，按本机最后阅读时间倒序显示最近100条记录。进入阅读器并成功显示首个页面时更新时间，同一次阅读会话此后最多每分钟更新一次，退出时再更新一次。不可访问或已删除的文件不显示。
   - 只有服务端明确返回某UUID不存在时，Android才自动删除该UUID的浏览进度和最近阅读记录；网络错误、服务端暂时不可用、鉴权失败或普通列表中未出现该UUID均不能触发删除。文件删除后重新添加获得新UUID，不继承旧进度。最近阅读超过100条时移除最旧的最近阅读记录及时间，但保留对应文件的页码进度；不提供手动清除入口。
   - 网络断开时，已经显示的缓存图片可以继续显示；未缓存内容显示“无法连接服务端”并提供重试。文件列表、搜索、详情、缩略图和原图等只读请求在网络错误后最多自动重试3次，间隔为1秒、2秒和5秒；网络恢复事件发生时，可立即重试当前界面仍需要且尚未成功的只读请求，离开相关界面后取消未执行的重试。
   - 评分接口采用“设置为指定分值”的幂等语义，可使用相同重试策略。鉴权失败、参数错误、文件不存在等明确的HTTP客户端错误不自动重试。自动重试耗尽后显示错误和手动“重试”按钮，手动重试开启新一轮最多3次自动重试。预读取失败不显示全局错误，只暂停对应页面的预读取；用户实际翻到该页时按当前页请求处理。
  - 有文件列表浏览、最近阅读、文件详情浏览、图片浏览、设置等基本视图。
  - 文件列表浏览界面采用纵向排列，每行是一个文件（或文件夹）。
   - 每行左侧是缩略图，设置中提供占屏幕宽度20%、30%和40%三档；缩略图保持宽高比完整显示，不裁切，行高由所选档位统一确定，避免滚动时高度跳动。没有可用封面的ZIP显示统一的“无封面”占位图，文件夹显示文件夹图标。
   - 右侧固定显示两行，第一行是文件名，第二行是tag摘要；摘要只显示作者、原作和语言tag，其余内容可在文件详情页的未分类tag中查看。两行超出时均使用省略号且不换行，点击ZIP后可在文件详情页查看完整文件名和全部tag。文件夹名称单行显示，超出时同样使用省略号。
   - 点击文件夹则进入下一层文件夹，点击文件则进入“文件详情浏览界面”
   - 列表顶部显示当前相对路径并提供返回上级目录按钮，路径中的每一级均可点击。进入下级目录时记录上级目录的筛选、排序和滚动位置；Android系统返回键优先返回上级目录并恢复位置，位于Content根目录时再次返回才退出当前页面或应用。跨多级点击路径时丢弃更深层导航记录。
   - 从文件详情或递归搜索结果返回时恢复原目录、筛选条件及滚动位置。刷新不清除筛选和排序；原位置超出新结果范围时定位到最接近的有效位置。
   - Android滚动到列表底部时自动加载下一页。
   - 支持文件名搜索、tag和评分筛选、范围选择及排序，规则与服务端列表接口一致。
  - 文件详情浏览界面主要是浏览当前文件的元数据和快速浏览内容缩略图。
   - Android和Web详情页均显示完整文件名、相对Content的路径、文件大小、文件修改时间、总页数及不可用页数量、作者/原作/语言tag、未分类tag、评分、处理状态及简短错误原因。Web额外显示UUID、设备号和inode、首次发现时间、最近分析时间、解析器版本、缩略图生成进度及缓存占用。
   - 文件大小使用易读单位，Web同时提供精确字节数；时间按客户端本地时区显示，Web可查看原始ISO时间。没有值的可选字段显示“未识别”，不隐藏整行；详细异常堆栈只写入日志。
   - 竖屏时顶部信息区约占首屏高度40%，左侧封面占该区域宽度40%，右侧显示文件名、tag、可点击评分和“从X页开始浏览”按钮。封面保持原始宽高比完整显示，不裁切；文件名和tag区域允许纵向滚动或折叠，不能挤压评分和开始阅读按钮。“约40%”是布局目标，可根据系统状态栏、字体缩放和安全区域调整，不作为固定像素验收值。
   - 横屏时改为左侧固定展示封面和元数据、右侧展示内容缩略图，不强制使用40%高度。内容缩略图区域随页面整体滚动，不创建内部嵌套滚动区域。
   - 下面是内容缩略图网格，Android设置提供每行4、5、6张三档，默认4张，保存在本机并应用于所有文件，不支持单文件覆盖；竖屏和横屏使用相同列数。缩略图完整包含且不裁切，单元格和序号区域尺寸固定，加载状态不得改变网格尺寸。修改列数后以原首个可见页码为锚点恢复大致滚动位置。点击图片后进入对应页的图片浏览界面。
   - 文件详情页只加载内容缩略图。内容缩略图尚未生成时显示生成状态并按当前可见范围优先生成。
   - 返回功能是回到“文件列表浏览”界面
  - 图片浏览界面是全屏显示图片。
   - 进入图片浏览界面后按需读取原始图片。原图加载完成前先显示内容缩略图；客户端按屏幕显示尺寸解码，放大时再提高解码精度，避免将超大图片直接完整解码到内存。服务端不预先解压保存全部原图。
   - 默认图片自适应屏幕大小。可双指缩放。双击放大图片到原始大小，再次双击恢复到自适应屏幕大小。图片处于自适应大小时，单指左右滑动用于翻页；放大后单指拖动只用于平移，即使到达图片边缘也不触发翻页，但点击屏幕左右侧仍可翻页，中央点击仍用于显示或隐藏控制栏。双击恢复自适应大小后，单指滑动重新用于翻页。
   - 翻页成功后下一页恢复默认的自适应屏幕大小；翻页请求失败时保留当前页及其缩放位置。第一页和最后一页不循环。
   - 设置中提供“从左向右”和“从右向左”两种阅读方向，默认从左向右。“下一页”始终表示页码加1，“上一页”始终表示页码减1。从左向右时，向左滑动或点击屏幕右侧进入下一页，向右滑动或点击左侧返回上一页；从右向左时，向右滑动或点击屏幕左侧进入下一页，向左滑动或点击右侧返回上一页。
   - 底部滑动条从左到右始终为第1页到总页数，不随阅读方向反转。到达第一页或最后一页时不翻页，可显示轻微边界反馈但不弹出提示。阅读方向是本地全局设置，不支持单文件覆盖。
   - 点击屏幕中央显示或隐藏屏幕下方的滑动条，实现快速翻页。不提供横屏/竖屏切换按钮，图片浏览界面跟随Android系统的旋转状态；屏幕旋转时保留当前页码，并将图片恢复为自适应大小，不保留缩放和平移位置。
   - 阅读器不主动轮询或订阅ZIP内容版本变化。图片读取期间如果ZIP变化导致条目不存在、CRC错误或其他读取失败，服务端返回明确的读取错误并记录详细日志；Android捕获错误，当前页显示“图片读取失败”并提供“重试”和“返回详情”，不得崩溃。重试仍失败时用户可返回详情页重新获取最新元数据和图片清单；已显示页面不受影响，预读取失败仍按既有规则静默停止对应页面。
  - 设置界面。内容包括有
   - 服务端地址、端口、APIkey设置（有测试连接按钮）
   - “测试连接”依次验证地址和端口格式、10秒内可连接、服务端返回Dokura身份及API版本、APIkey鉴权成功，以及客户端支持该API版本；全部通过才显示连接成功和服务端版本。失败时区分地址无效、连接超时、非Dokura服务、APIkey无效和版本不兼容，提示及日志不得回显APIkey。测试不启动扫描、生成缩略图或修改服务端状态。保存设置与测试分开，允许保存暂时无法连接的配置。
   - 显示当前缓存用量、缓存上限和清空缓存按钮。
   - 主题提供“跟随系统、亮色、暗色”三种模式，默认跟随系统，只保存在当前Android设备。跟随系统时系统主题变化立即生效；切换主题不重新加载页面，也不影响筛选、阅读页码或表单内容。主题只影响界面控件和背景，不对图片进行颜色反转或亮度处理。
   - “阅读时保持屏幕常亮”开关，默认关闭。开启后仅在图片浏览界面保持常亮，离开后恢复Android系统原有的息屏行为。
   - 版本号信息。
   - 打开项目github（通过浏览器）
