# 项目简介
 - Dokura，一个适用于管理Nas上大量同人志，并可以在安卓apk应用中快速浏览的项目。
 - 关键词：轻量化，高性能，无需外网

# 项目架构
 ## 服务端
  - 长时运行的后台服务。实现文件监控处理以及管理用的web界面
  
 ## 客户端
  - 安卓apk应用。连接服务端，浏览同人志。

# 项目需求
 ## 基础需求边界
  - 同人志是大量.zip文件，内含大量png或jpg图片。在安卓的客户端端需要实现高性能的实时浏览。
  - 需管理的同人志数量是10000~100000个zip文件的级别。
  - 每个zip文件的大小约为10MB ~ 1000MB
  - 客户端与服务端之间的通信采用（待考虑）方式，提供局域网100MB级别的传输。
  - 默认是单人使用，不需要多人协作功能。评分在服务端仅保存1份，供安卓客户端和Web端共享；浏览进度仅保存在安卓客户端本地，不跨设备同步，Web端不保存浏览进度。
  - 首版仅支持在线连接服务端浏览，客户端缓存只用于在线访问加速，不提供完整的离线书库功能。

 ## 服务端（后台）
  - 服务端不需要实现从外网下载同人志资源。
  - 文件发现和同步：
   - 服务启动时执行一次增量扫描，运行期间监听Content目录中的新增、删除、移动、重命名和修改事件。Web设置页提供“立即扫描”按钮。
   - 忽略文件名以`!.qB`结尾的下载临时文件。仅在ZIP文件的大小和修改时间稳定后开始处理，避免读取尚未写完的文件。
   - 使用文件路径、大小和修改时间判断文件变化。文件被外部删除后立即从列表隐藏，遗留元数据和缓存由“清理缓存”功能回收。
   - 文件处理失败后记录原因并有限次数自动重试；文件没有变化时不重复处理，大小或修改时间变化后自动重试。
  - 文件身份：
   - 每个ZIP文件记录使用服务端生成的随机UUID作为唯一ID，不计算SHA-256。内容相同但路径不同的文件是两个独立文件；文件删除后再次添加时分配新的UUID。
   - 运行期间的移动或重命名通过文件监听事件识别并保留UUID。同一文件系统内，服务重启扫描时使用设备号、inode和文件大小识别停机期间发生的移动或重命名。跨文件系统移动按删除旧文件并新增文件处理。
   - 路径和inode不变而大小或修改时间变化时，视为原文件更新，保留UUID和评分并重新分析；同一路径但inode变化时，视为文件替换并分配新UUID。
  - ZIP内容处理：
   - 首版仅支持未加密的标准ZIP，以及`.jpg`、`.jpeg`、`.png`图片，图片扩展名大小写不敏感。
   - 图片按ZIP内相对路径自然排序，目录层级优先，同级文件名按`1、2、10`的顺序排列。忽略目录项、隐藏文件、`__MACOSX`和非图片文件。
   - 排序后的第一张有效图片作为封面。ZIP没有有效图片时标记为“无有效内容”。
   - 损坏、无法读取或需要密码的ZIP仍显示在文件列表中，并标记为“处理失败”。列表显示简短原因，详细异常写入日志，Web文件详情页提供“重新处理”命令。
   - 单张图片解压失败时跳过该图片并记录警告，其余图片仍可阅读。检查ZIP条目的解压后大小，拒绝明显异常的压缩炸弹。
  - 新文件发现后立即解析文件名tag、图片清单和封面，并保存相关元数据。
  - 缩略图包括有两种：“封面缩略图”和“内容缩略图”
   - “封面缩略图”是排序后的第一张有效图片，按原分辨率保存，并在首次扫描时立即生成。
   - “内容缩略图”是ZIP内图片最大不超过1280*720的jpg缩略图。首次打开文件详情页时按需进入生成队列，优先生成当前可见范围，随后在后台完成该ZIP的其余缩略图。
   - 内容缩略图持久保存在MetaData目录。Web设置页显示生成任务进度，并提供暂停、继续和取消等待任务的功能。默认仅运行1个生成任务，避免占满NAS的I/O和CPU。
   - ZIP更新后旧封面和内容缩略图立即失效，由缓存清理功能回收；新内容在扫描或再次访问时生成。
  - tag只从文件名解析，不需要连接网络源，也不提供手动tag功能。解析规则见`/docs/tags-rules.md`。
  - 所有文件列表、搜索和筛选接口必须服务端分页，默认每页50条，最大每页200条。
   - 默认按文件名自然升序排列，文件夹始终排在文件之前。
   - 支持按文件名、文件大小、修改时间和评分排序，并支持升序和降序。
   - 文件名搜索、tag筛选和评分筛选可以组合，多种条件之间采用“并且”关系。多个tag可选择“全部匹配”或“匹配任一”。
   - 搜索和筛选范围可选择“仅当前文件夹”或“包含子文件夹”，默认仅当前文件夹。
  - docker挂载3个文件夹：
   - MetaData文件夹：用于储存各ZIP文件的元数据、UUID和评分
   - Config文件夹：储存服务端配置、管理员凭据、APIkey和日志
   - Content文件夹：存放和管理同人志zip文件的文件夹
  - MetaData和Config必须使用持久化卷，Docker更新或重建不得删除。首版不提供应用内备份和恢复功能，文档要求用户通过NAS备份工具同时备份同一时间点的MetaData和Config；缩略图缓存可以不备份。
  - 服务端和安卓客户端之间采用APIkey鉴权。安卓客户端不能调用删除、移动、重命名和后台处理控制等管理接口。
  - APIkey支持随机重新生成，生成后旧key立即失效。
  - 后台处理支持暂停和继续。暂停只停止文件监听、增量扫描、缩略图生成和失败重试；当前单个任务完成后再暂停。Web管理、安卓浏览、搜索、评分和读取已有图片继续可用。恢复后通过增量扫描补齐暂停期间的文件变化。
  - 生成日志文件并存放在Config卷中。单个日志文件最大20MB，保留最近10个文件。日志不得记录密码、APIkey等鉴权信息；正常的单张图片请求不记录信息级日志。

 ## 客户端（安卓APK应用）
  - 客户端仅需求安卓apk版本，不需要考虑apple版本。
  - 特殊功能：
   - 缓存功能：缓存封面缩略图、内容缩略图和已读取的原图，默认上限为5GB。设置提供1GB、5GB、10GB、20GB四档。达到上限后按最近最少使用顺序依次淘汰原图、内容缩略图和封面，当前正在查看及预读取队列中的图片暂不淘汰。
   - 清空缓存只删除图片缓存，不删除服务端连接配置、浏览进度、最近阅读和主题设置。执行前显示缓存用量并要求确认。
   - 预读取功能：进入“文件详情浏览”后，从服务端预读取当前页之前3页和之后10页的原始图片。设备可用空间不足时停止预读取，但仍允许按需加载当前页。
   - 评分功能：用户对当前zip文件评分，为0~5星（默认为0星）。该评分保存在服务器端。
   - 浏览进度：按文件UUID在本地记录当前浏览页码和最后阅读时间，下次直接跳转，不跨设备同步。文件内容更新后若记录页码超过新的总页数，则调整到最后一页。
   - 最近阅读：在文件列表页提供最近阅读视图，按本机最后阅读时间倒序显示最近100条记录。不可访问或已删除的文件不显示。清除浏览进度时同时清除最近阅读记录。
   - 网络断开时，已经显示的缓存图片可以继续显示；未缓存内容显示“无法连接服务端”并提供重试。文件列表、搜索、详情和评分要求连接服务端，短暂中断后自动恢复请求。
  - 有文件列表浏览、最近阅读、文件详情浏览、图片浏览、设置等基本视图。
  - 文件列表浏览界面采用纵向排列，每行是一个文件（或文件夹）。
   - 每行的左侧是缩率图（可设置占屏幕的20%，30%，40%三种宽度）
   - 右侧是文件名和tags分两行显示，第一行是文件名，第二行是tag。内容溢出屏幕时不换行（可在“单文件夹浏览界面”查看）。
   - 点击文件夹则进入下一层文件夹，点击文件则进入“文件详情浏览界面”
   - Android滚动到列表底部时自动加载下一页。
   - 支持文件名搜索、tag和评分筛选、范围选择及排序，规则与服务端列表接口一致。
  - 文件详情浏览界面主要是浏览当前文件的元数据和快速浏览内容缩略图。
   - 页面上40%屏幕高度为左侧封面缩略图，右侧是文件名，tags，评分（评分可点击），从x页开始浏览按钮（根据浏览进度记录）。
   - 下面是内容缩略图的平铺，可设定每行4~6个图片。图片下面带有序号。点击图片后进入该图片的“图片浏览界面”。
   - 文件详情页只加载内容缩略图。内容缩略图尚未生成时显示生成状态并按当前可见范围优先生成。
   - 返回功能是回到“文件列表浏览”界面
  - 图片浏览界面是全屏显示图片。
   - 进入图片浏览界面后按需读取原始图片。原图加载完成前先显示内容缩略图；客户端按屏幕显示尺寸解码，放大时再提高解码精度，避免将超大图片直接完整解码到内存。服务端不预先解压保存全部原图。
   - 默认图片自适应屏幕大小。可双指缩放。双击放大图片到原始大小，再次双击恢复到自适应屏幕大小。翻页后恢复默认的自适应屏幕大小。
   - 可通过左右滑动或点击屏幕左右侧实现翻页，第一页和最后一页不循环。
   - 设置中提供“从左向右”和“从右向左”两种阅读方向，默认从左向右。底部滑动条的页码顺序始终为1到总页数，不随阅读方向反转。该设置为本地全局设置，不支持单文件覆盖。
   - 点击屏幕中央唤出滑动条在屏幕下方，实现快速翻页。滑动条旁边有横屏/竖屏按钮
   - 正在阅读时如果服务端发现ZIP内容更新，提示内容已更新并重新加载详情，不继续使用旧的页码清单。
  - 设置界面。内容包括有
   - 服务端地址、端口、APIkey设置（有测试连接按钮）
   - 显示当前缓存用量、缓存上限和清空缓存按钮。
   - 亮色/暗色主题切换
   - “阅读时保持屏幕常亮”开关，默认关闭。开启后仅在图片浏览界面保持常亮，离开后恢复Android系统原有的息屏行为。
   - 清除浏览进度和最近阅读记录。
   - 版本号信息。
   - 打开项目github（通过浏览器）
   

 ## 服务端（web界面）
  - web界面采用账号密码登录，默认账号admin，密码admin。首次登录后醒目提示用户修改默认密码，但不强制修改。
   - 密码仅保存强哈希，不保存明文。Web登录会话有过期时间，退出后立即失效；连续登录失败时进行短时限流。
   - 不提供邮箱找回密码，通过Docker命令重置管理员密码。
  - web界面有类似安卓客户端的“文件列表浏览界面”，“文件详情浏览界面”，“图片浏览界面”，“设置界面”，“后台日志”，但不记录浏览进度，也不提供最近阅读和阅读方向设置。
  - “文件列表浏览界面”，“文件详情浏览界面”，“图片浏览界面”三个界面的功能与客户端基本一致，除此之外，还有额外功能如下：
   - Web使用常规分页控件。支持跨分页选择完整筛选结果，而不是只选择当前已加载页。
   - 文件支持删除、移动和重命名；支持多选ZIP后批量移动。首版不支持批量重命名、上传ZIP、解压ZIP或创建压缩包。
   - 文件夹支持新建、重命名和移动。禁止将文件夹移入自身或其子目录；只允许删除空文件夹，含ZIP或子目录的文件夹不能直接删除。
   - 移动只能选择Content内已有文件夹作为目标。重命名必须保留`.zip`扩展名，禁止空名称、`.`、`..`、路径分隔符及任何可能越过Content根目录的输入。
   - 目标路径存在同名文件或文件夹时操作失败，绝不覆盖。批量移动前检查全部冲突，存在任何冲突时不执行并列出冲突项。成功移动或重命名后保留UUID和评分。
   - 删除为永久删除，不提供回收站。批量删除前由服务端重新计算完整筛选结果的文件数量和总大小，在界面显示并要求二次确认。
   - 必须支持“筛选出3星及以下文件（含0星），跨分页全选并批量删除”的流程。
  - “设置界面”的功能有：
   - 后台处理暂停/继续按钮，并显示后台处理状态和等待任务数量。Docker容器和FastAPI服务只能通过Docker或NAS管理界面启停。
   - APIkey设置（包括随机生成按钮）
   - 显示当前缓存用量。
   - 清理缓存功能（扫描已经被删除的zip文件，删除相应的元数据和缓存数据）
   - 版本号信息
   - 亮色/暗色主题切换
   - 项目github超链接
  - “后台日志”界面：
   - 默认显示当前日志最近1000条，支持按错误、警告、信息级别筛选。
   - 支持将当前和已轮转日志打包下载，不提供从Web删除或修改日志的功能。
