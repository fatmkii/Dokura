# 项目简介
 - Dokura，一个适用于管理Nas上大量同人志，并可以在安卓apk应用中快速浏览的项目。
 - 关键词：轻量化，高性能，无需外网

# 项目架构
 ## 服务端
  - 长时运行的后台服务。实现文件监控处理以及管理用的web界面
  
 ## 客户端
  - 安卓apk应用。连接服务端，浏览同人志。

# 项目需求
 ## 基础需求边界
  - 同人志是大量.zip文件，内含大量png或jpg图片。在安卓的客户端端需要实现高性能的实时浏览。
  - 需管理的同人志数量是10000~100000个zip文件的级别。
  - 每个zip文件的大小约为10MB ~ 1000MB
  - 客户端与服务端之间的通信采用（待考虑）方式，提供局域网100MB级别的传输。
  - 默认是单人使用，不需要多人共用的功能设计（例如评分、浏览进度仅需1套）。

 ## 服务端（后台）
  - 服务端不需要实现从外网下载同人志资源。
  - 要自动发现并扫描文件，保存缩略图和tag等元数据。发现新文件后后台即刻开始读取文件分析tag和生成缩略图。
  - 缩略图包括有两种：“封面缩略图”和“内容缩略图”
   - “封面缩略图”是zip文件的第一个图片，按原分辨率保存
   - “内容缩略图”是zip文件内所有图片的缩略图，但分辨率压缩为最大不超过1280*720的jpg图片后再保存。
  - tag是从文件名解析的，不需要连接网络源进行解析。解析的规则见`/docs/tag-ruls.md`
  - docker挂载3个文件夹：
   - MetaData文件夹：用于储存各zip文件的元数据
   - Config文件夹：储存服务端相关配置
   - Content文件夹：存放和管理同人志zip文件的文件夹
  - 服务端和客户端之间采用APIkey鉴权。
  - 生成日志文件

 ## 客户端（安卓APK应用）
  - 客户端仅需求安卓apk版本，不需要考虑apple版本。
  - 特殊功能：
   - 缓存功能：包括封面缩略图和文件内容。设置中有清空缓存功能。
   - 预读取功能：进入了“文件详情浏览”后，从服务端预读取前3页和后10页的内容。
   - 评分功能：用户对当前zip文件评分，为0~5星（默认为0星）。该评分保存在服务器端。
   - 浏览进度：记录当前zip文件的浏览进度在第几页。下次直接跳转。浏览进度仅保存在客户端。
  - 有文件列表浏览、文件详情浏览，图片浏览、设置三个基本界面
  - 文件列表浏览界面采用纵向排列，每行是一个文件（或文件夹）。
   - 每行的左侧是缩率图（可设置占屏幕的20%，30%，40%三种宽度）
   - 右侧是文件名和tags分两行显示，第一行是文件名，第二行是tag。内容溢出屏幕时不换行（可在“单文件夹浏览界面”查看）。
   - 点击文件夹则进入下一层文件夹，点击文件则进入“文件详情浏览界面”
   - 有搜索功能：根据文件名关键词搜索，搜索是搜索当前文件夹和下级所有文件夹的。
   - 有筛选功能：根据tags和评分筛选，筛选只适用于当前文件夹的。
  - 文件详情浏览界面主要是浏览当前文件的元数据和快速浏览内容缩略图。
   - 页面上40%屏幕高度为左侧封面缩略图，右侧是文件名，tags，评分（评分可点击），从x页开始浏览按钮（根据浏览进度记录）。
   - 下面是内容缩略图的平铺，可设定每行4~6个图片。图片下面带有序号。点击图片后进入该图片的“图片浏览界面”。
   - 返回功能是回到“文件列表浏览”界面
  - 图片浏览界面是全屏显示图片。
   - 默认图片自适应屏幕大小。可双指缩放。双击是放大图片到原始大小，再次双击恢复到自适应屏幕大小。翻页后维持默认的自适应屏幕大小。
   - 可通过左右滑动或点击屏幕左右侧实现翻页
   - 点击屏幕中央唤出滑动条在屏幕下方，实现快速翻页。滑动条旁边有横屏/竖屏按钮
  - 设置界面。内容包括有
   - 服务端地址、端口、APIkey设置（有测试连接按钮）
   - 显示当前缓存用量和清空缓存按钮。
   - 亮色/暗色主题切换
   - 息屏时间：默认5s。可设置-1则为永不息屏。
   - 版本号信息。
   - 打开项目github（通过浏览器）
   

 ## 服务端（web界面）
  - web界面采用账号密码登录，默认账号admin，密码admin
  - web界面有类似上述客户的“文件列表浏览界面”，“文件详情浏览界面”，“图片浏览界面”，“设置界面”，“后台日志”。
  - “文件列表浏览界面”，“文件详情浏览界面”，“图片浏览界面”三个界面的功能与客户端基本一致，除此之外，还有额外功能如下：
   - “文件列表浏览界面”：要有文件的删除、移动、重命名功能。有多选和筛选功能。要能够实现这个流程“在当前文件夹下筛选出3星及以下文件（含0星），并全选批量删除”
  - “设置界面”的功能有：
   - 服务启动/关闭按钮
   - APIkey设置（包括随机生成按钮）
   - 显示当前缓存用量。
   - 清理缓存功能（扫描已经被删除的zip文件，删除相应的元数据和缓存数据）
   - 版本号信息
   - 亮色/暗色主题切换
   - 项目github超链接
  - “后台日志”界面：显示当前日志和下载日志的功能。
