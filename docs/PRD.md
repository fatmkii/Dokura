# Dokura共通需求

本文档定义项目的共通需求、系统边界和跨端约束。

## 项目定位

 - Dokura，一个适用于管理Nas上大量同人志，并可以在安卓apk应用中快速浏览的项目。
 - 关键词：轻量化，高性能，无需外网

### 语言与本地化

 - 首版Android、Web界面和项目文档只提供简体中文，不实现语言切换或自动翻译。文件名、tag和ZIP内路径保留原始Unicode文本，不翻译或转写；日期、文件大小和数字按简体中文格式显示。界面文案在代码中集中管理，避免散落硬编码。

## 系统架构

### 服务端

  - 长时运行的后台服务。实现文件监控处理以及管理用的web界面

### 客户端

  - 安卓apk应用。连接服务端，浏览同人志。

## 产品范围与规模

### 内容形态

  - 同人志是大量.zip文件，内含大量png或jpg图片。在安卓的客户端端需要实现高性能的实时浏览。

### 数据规模

  - 首版按10,000至100,000个ZIP的规模设计和验收，该范围不是硬数量上限；超过100,000后仍保持功能正确并继续发现文件，但不承诺达到首版性能目标。数量按当前Content中可见ZIP计算，不包含已删除但尚未清理的元数据。性能验收使用包含100,000个ZIP元数据、接近真实目录层级和tag分布的数据集；列表、搜索和分页可用少量真实ZIP配合代表性元数据及文件结构测试，真实ZIP处理性能另行测试。
  - 每个ZIP文件的预期大小约为10MB至1000MB，该范围是容量估算而非导入限制；小于10MB或大于1000MB的有效ZIP仍正常处理，首版不按压缩包文件大小设置硬上限。安全硬限制只采用ZIP内容处理章节规定的单条目解压大小、总解压大小和压缩比规则。性能测试至少覆盖接近1GB的ZIP，但不承诺任意超大文件具有相同响应速度。

### 用户与在线边界

  - 默认是单人使用，不需要多人协作功能。评分在服务端仅保存1份，供安卓客户端和Web端共享；浏览进度仅保存在安卓客户端本地，不跨设备同步，Web端不保存浏览进度。
  - 首版仅支持在线连接服务端浏览，客户端缓存只用于在线访问加速，不提供完整的离线书库功能。

## 通信与图片传输

  - 客户端与服务端之间采用HTTP/1.1 REST API通信，API以`/api/v1/`为前缀并通过OpenAPI描述。结构化数据使用JSON；图片和日志归档等文件使用独立HTTP接口返回二进制，不使用JSON或Base64封装。后台状态采用定时轮询，首版不使用WebSocket、SSE或gRPC。
  - 图片响应支持ETag和缓存控制；已落盘且可直接读取的封面缩略图支持HTTP Range并采用高效文件响应。详情网格使用服务端按请求尺寸即时生成的预览图，服务端不预先生成或永久保存；图片浏览使用ZIP内原图，服务端边解压边流式返回。在千兆局域网且服务端存储、CPU及客户端不构成瓶颈的条件下，大文件连续传输目标达到100MB/s级别；首次从NAS读取并解压或缩放ZIP内图片不承诺达到该吞吐量。

## 性能验收

  - 首版参考验收环境为4核CPU、8GB内存、MetaData位于本地SSD、Content位于顺序读取不低于150MB/s的NAS存储、服务端与客户端使用千兆局域网；数据集包含100,000条元数据和接近真实分布的目录、tag及文件名，并使用10MB至1GB的真实ZIP、2MB至20MB且不超过4000万像素的代表性JPEG/PNG验证图片路径。验收报告必须记录实际CPU、内存、存储类型、网络条件、数据集、冷缓存或热缓存状态及测试版本。
  - 在单用户、后台扫描未运行且存储已唤醒的参考环境中：已建立索引的列表、搜索、筛选和排序API的95%请求在500毫秒内返回；Android和Web打开列表或详情且已有封面缩略图时，元数据和封面首屏在1秒内可操作；详情页可见预览图在300毫秒内显示固定占位，服务端在1秒内开始返回且95%的代表性预览图在2秒内完整显示；未缓存的当前页原图在1秒内开始返回且95%的代表性图片在3秒内完成适屏显示；Android磁盘缓存中的原图在翻页后300毫秒内显示；评分界面在300毫秒内乐观更新，正常局域网下1秒内获得服务端确认。
  - 性能验收必须同时覆盖：冷启动后的首次列表与首次图片读取、热缓存重复访问、后台完整扫描100,000个ZIP元数据期间持续浏览、连续阅读60分钟、Android缓存接近20GB上限及主动清空缓存。后台扫描期间，前台列表、搜索和图片开始返回时间的95%不得超过对应空闲指标的2倍，评分仍须在1秒内获得确认；完整对账扫描在参考环境中应在10分钟内完成，不包含首次逐个ZIP分析和NAS休眠唤醒时间。
  - 服务端以连续60分钟“后台扫描加前台浏览”作为资源稳定性验收：进程空闲稳定5分钟后的RSS不超过300MB，测试期间RSS峰值不超过1.5GB，最后10分钟的稳定值不得比最初10分钟高10%以上；打开的ZIP句柄、文件描述符、工作队列和线程数量均不得随请求总数持续增长。Android连续阅读60分钟不得出现内存不足、明显持续掉帧或因图片尺寸增长而无界增加位图内存。
  - 上述时间指标不包含首次ZIP分析、首次封面缩略图生成、NAS休眠唤醒或网络中断恢复；这三类场景仍须单独记录耗时、峰值资源和恢复结果。若参考硬件未达到目标，应先通过代表性基准定位瓶颈，再调整实现或经评审修改指标，不能静默降低验收标准。

## UI设计参考使用原则

 - 本PRD列出的开源项目只用于参考信息层级、页面布局、交互反馈、加载与错误状态以及阅读器样式，不因此引入参考项目的系列、推荐、多用户、在线内容源、下载、追踪、插件或其他未列入首版范围的功能。参考项目与本PRD冲突时，以本PRD的明确需求为准。
 - 可以借鉴通用视觉和交互模式，不要求复刻任何项目。直接复用源码、样式、图标或其他资产前必须核对其许可证、版权和署名要求；不能仅因项目开源就直接复制。
 - Web与Android应保持相同的状态码含义、中文业务术语、评分样式和“页面不可用”等核心状态表达，但导航结构、控件密度和阅读器交互按各自平台设计，不要求像素级一致。

## 技术栈共通约束

 - 首版开发基线为Python 3.14.x、FastAPI 0.138.x、Vue 3.5.x、Naive UI 2.44.x、Kotlin 2.4.x、Jetpack Compose stable BOM 2026.06.x、Android Gradle Plugin 9.3.x、Node.js 24.x LTS、TypeScript 5.9.x和Vite 7.x；Android构建使用JDK 17和与Android Gradle Plugin兼容的Gradle 9.5.x。
 - PRD记录兼容的版本系列，实际构建使用的精确版本由锁文件或构建配置记录：Python依赖使用`uv.lock`，Web依赖使用包管理器锁文件，Android依赖使用Gradle Version Catalog和Compose BOM、Gradle版本使用Wrapper固定。Docker基础镜像固定到补丁版本和镜像摘要。开发期间可在同一minor系列内升级patch版本；minor或major升级必须单独完成兼容性验证。
