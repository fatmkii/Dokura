# Dokura 首版分阶段实施计划

## 总体方案

采用单仓库结构：

- `server/`：Python 3.14、FastAPI、SQLAlchemy、Alembic、SQLite。
- `web/`：Vue 3、TypeScript、Vite、Naive UI；生产构建由 FastAPI 同源托管。
- `android/`：Kotlin、Jetpack Compose、Retrofit/OkHttp、Room、DataStore。
- `scripts/acceptance/`：各阶段一键验收脚本、测试数据生成器和性能报告工具。
- Docker 单服务进程、三个持久化挂载：`Content`、`MetaData`、`Config`。

每阶段必须通过对应验收脚本才能进入下一阶段。测试只读使用现有 `content/` 中的 201 个 ZIP；破坏性、异常和文件管理测试使用临时副本。

## 公共接口与核心模型

- 所有业务 API 使用 `/api/v1/`，FastAPI OpenAPI 文件纳入版本控制并执行漂移检查。
- Android 使用 `Authorization: Bearer <APIkey>`；Web 使用 HttpOnly、SameSite 会话 Cookie，管理写操作校验 Origin。
- 核心接口分组：
  - 健康与身份：服务可用、扫描状态、Dokura 身份及 API 版本。
  - 目录与文件：分页列表、搜索、筛选、排序、tag 候选、详情、评分。
  - 图片：封面、指定档位预览图、原图流式读取，支持 ETag、缓存控制及规定的 Range。
  - 管理：登录、密码、APIkey、扫描、任务、重新处理、缓存清理、日志、文件管理及选择快照。
- SQLite 至少包含文件、页面、tag 关联、任务、扫描记录、Web 会话和缓存索引；状态码、UUID、内容版本、解析器版本和不可用页面结论均持久化。
- Android Room 保存进度、最近阅读和缓存索引；DataStore 保存连接、主题、阅读方向及显示设置。
- 中文 UI 文案、状态码映射和业务术语集中管理，不散落硬编码。

## 分阶段实施

### 阶段 0：工程骨架与契约基线

实施内容：

- 建立 Server、Web、Android、Docker Compose 和验收脚本目录。
- 使用 `uv.lock`、`package-lock.json`、Gradle Wrapper、Version Catalog 和 Compose BOM 固定 PRD 规定的版本系列。
- 配置 SQLite 3.51.3、FTS5 trigram 启动自检，以及 Content/MetaData/Config 路径配置。
- 建立统一错误响应、日志脱敏、中文文案目录和 OpenAPI 版本规则。
- 建立单元、集成、Web E2E、Android JVM/仪器测试入口。

验收方式：

- `scripts/acceptance/stage-00.sh` 完成 Server 测试、Web 类型检查与构建、Android lint/JVM 测试及 debug APK 构建。
- Docker 启动后健康接口立即可用，不等待扫描；三个目录重建容器后数据仍保留。
- 启动时实际验证 SQLite 版本和 FTS5；模拟不满足条件时必须拒绝启动并给出明确原因。
- OpenAPI 可生成且重复生成无未提交差异。

### 阶段 1：元数据、文件名解析与安全 ZIP 分析

实施内容：

- 完成 Alembic 初始模型、WAL/外键/FULL synchronous/busy timeout、最多五个读连接和单写者调度。
- 按 `tags-rules.md` 实现状态机式文件名解析、Unicode NFC/casefold、自然排序键、分类 tag、未分类 tag、置信度、警告和解析器版本。
- 实现 ZIP 中央目录解析、条目过滤与排序、路径穿越防护、加密/损坏识别、解压大小和压缩比限制、图片尺寸预检。
- 生成最长边 720px、质量 85 的持久化封面；ZIP 与图片处理期间不持有数据库事务。
- 建立确定性页面不可用和普通暂时读取失败的区分。

验收方式：

- `scripts/acceptance/stage-01.sh` 执行解析器、自然排序、Unicode、恶意 ZIP、CRC、像素炸弹及事务边界测试。
- `tags-rules.md` 的规定样例和异常括号样例逐字段断言通过，原始文件名不被改写。
- 无图片 ZIP、损坏 ZIP、加密 ZIP、安全限制 ZIP 分别进入规定状态；单页损坏不改变页码和总页数。
- 对现有 201 个 ZIP 进行只读分析报告，ZIP 全部被处理或得到可解释状态，非 ZIP 文件被忽略，样例目录零修改。
- 中途修改 ZIP 的测试证明旧元数据不被半成品覆盖，临时封面不会成为有效缓存。

### 阶段 2：扫描、监听、身份与后台任务

实施内容：

- 实现启动增量扫描、手动完整对账、文件系统监听和一秒事件合并。
- 使用集中式到期调度器实现 5 秒检查、连续三次稳定、10 秒/1 分钟/5 分钟重试。
- 落实 UUID、inode、设备号、路径、大小和修改时间的移动、重命名、替换与更新规则。
- 实现持久化任务恢复、去重、优先级、等待老化、单分析并发和前台压力让路。
- 实现不可访问目录的保护性对账、扫描状态、确认删除和服务端缓存清理预检/执行。

验收方式：

- `scripts/acceptance/stage-02.sh` 在临时目录自动执行新增、连续写入、修改、删除、同盘移动、重命名、替换、重启和重试场景。
- 验证同一 UUID 不出现并行分析；服务重启后任务和重试额度正确恢复。
- 根目录或子目录不可访问时不得误删元数据、UUID、评分或封面；恢复后原记录重新显示。
- 扫描期间重复触发只增加一次后续扫描；符号链接、非 ZIP 和临时扩展名不进入任务。
- 目录遍历采用流式、批量查询；测试查询计数证明不存在逐文件全表查询或 N+1。

### 阶段 3：只读 API、搜索、图片调度与鉴权

实施内容：

- 完成目录/文件分页、搜索、tag/评分筛选、范围选择和稳定排序。
- 建立 NFC/casefold 字段、FTS5 trigram、tag 关联和常用组合索引。
- 完成详情、评分、封面、256/512/768 预览图和原图流式接口。
- 实现图片用途优先级、并发槽位、64/32 队列限制、预览请求合并、128MB 内存 LRU 和 32 个 ZIP 句柄上限。
- 完成 Web 登录限流、会话生命周期、密码修改/重置和 Android APIkey 生成、轮换及权限隔离。

验收方式：

- `scripts/acceptance/stage-03.sh` 执行 OpenAPI 合约、权限、分页一致性、筛选组合、评分幂等和图片 HTTP 语义测试。
- 生成 100,000 条代表性元数据；常用查询通过 `EXPLAIN QUERY PLAN` 确认使用目标索引且无 N+1。
- 搜索精确满足 NFC 后不区分大小写的字面子串语义；1–2 字符查询和 trigram 查询返回一致语义。
- ETag/304、预览尺寸、JPEG 参数、Range、取消、超限 `Retry-After` 和内容版本失效测试通过。
- Android 凭据不能调用管理接口；日志、URL、异常和 OpenAPI 示例均不泄露密码、会话或 APIkey。

### 阶段 4：Web 浏览与阅读闭环

实施内容：

- 实现登录、纵向目录列表、面包屑、分页、URL 状态、搜索、筛选、排序和评分乐观更新。
- 实现文件详情、元数据分组、可见范围预览图网格和独立全屏阅读路由。
- 实现亮色、暗色、跟随系统主题，以及稳定的加载、空结果、失败和页面不可用状态。
- 保持轻量 NAS 内容库视觉，不引入系列、推荐、多用户或在线来源。

验收方式：

- `scripts/acceptance/stage-04.sh` 启动 Docker 后运行 Playwright Chromium E2E。
- 覆盖登录、目录返回状态恢复、300ms 搜索防抖、URL 刷新恢复、筛选排序、评分失败回滚和删除文件错误页。
- 详情页仅请求可见预览图；离屏未开始请求被取消，进入阅读器后才请求原图。
- 键盘方向键、Home、End、Esc、边界禁用和浏览器尺寸变化行为符合 PRD。
- 亮暗主题、200% 字体缩放、窄屏和桌面宽屏下无关键控件遮挡或布局跳动。

### 阶段 5：Web 管理闭环

实施内容：

- 实现跨分页选择快照、文件删除/移动/重命名、文件夹新建/移动/重命名/空目录删除。
- 实现名称、路径长度、Windows 保留名、同盘、冲突、真实路径和五秒互斥等待校验。
- 实现单文件重新处理、重试所有失败、重新扫描、任务轮询、APIkey 管理和缓存清理。
- 实现日志轮转、筛选和归档下载。

验收方式：

- `scripts/acceptance/stage-05.sh` 使用临时 Content 树执行完整管理 E2E，绝不操作真实样例。
- 验证“筛选 0–3 星、跨分页全选、二次确认、批量永久删除”流程及统计变化重新确认。
- 批量移动部分冲突时成功项保留、失败项可解释且不覆盖；目录移动保留后代 UUID、评分和关联缓存。
- 非空目录、符号链接目录、跨文件系统移动、路径逃逸、忙碌超时和竞态删除均被安全拒绝。
- 扫描、清理和任务页面轮询符合单实例及离页停止规则；日志轮转后总文件数不超过十个且归档内容完整。

### 阶段 6：Android 连接、目录、详情与本地状态

实施内容：

- 建立 Compose 导航、Retrofit/OkHttp、Room、DataStore 和系统安全凭据存储。
- 实现连接设置与无副作用测试连接、目录列表、无限分页、最近阅读、搜索筛选排序及状态恢复。
- 实现详情布局、评分、可见范围预览图网格和 4/5/6 列设置。
- 实现统一的 1/2/5 秒只读请求重试、网络恢复触发和离页取消。

验收方式：

- `scripts/acceptance/stage-06.sh` 运行 JVM 测试、Compose 测试并在 Windows `emulator-5554` 上执行仪器测试。
- 覆盖五类测试连接结果，确认任何日志和界面都不回显 APIkey。
- 验证目录层级返回、滚动恢复、分页版本变化 UUID 去重、300ms 防抖和过期响应忽略。
- 竖屏、横屏、主题切换、字体缩放和 4/5/6 列网格行为符合 PRD。
- 只有服务端明确返回 UUID 不存在时才移除本地最近阅读；网络、鉴权或服务暂时失败不删除状态。

### 阶段 7：Android 阅读器、预读取与磁盘缓存

实施内容：

- 实现全屏阅读器、当前页优先请求、左右阅读方向、手势、缩放、适屏、页码滑动条和旋转恢复。
- 实现采样解码及区域/分块解码，严格执行普通/低内存设备预算。
- 实现前 3 后 10 的预读取窗口、最多三个请求及当前页保留槽位。
- 实现 1/5/10/20GB 缓存、类别优先 LRU、保护集合、临时文件、原子替换、索引对账和清空缓存。
- 实现页面稳定显示 500ms 后的进度，以及最近阅读的每分钟/退出更新规则。

验收方式：

- `scripts/acceptance/stage-07.sh` 在模拟器运行手势、旋转、缓存、网络中断、进程重启和内存压力测试。
- 放大后拖动不翻页，边缘点击仍翻页；阅读方向仅改变手势映射，不反转页码条或预读取页码。
- 当前页能够抢占最远预读取；离开阅读器后队列取消且不留下可用的半文件。
- 缓存达到上限后按“临时文件→预览图→原图→封面”规则处理，受保护文件不删除，累计误差不超过 1MB。
- 清空缓存不删除连接、APIkey、评分、进度、最近阅读和设置；异常退出后的缓存索引可自动对账恢复。
- 超大图片不突破位图预算；内存压力下释放顺序正确且当前页面不崩溃。

### 阶段 8：部署、故障恢复与安全加固

实施内容：

- 完成固定补丁版本及摘要的生产 Docker 镜像、非特权运行、卷权限、健康检查和启动迁移。
- 验证 MetaData/Config 冷备份恢复流程、Content 恢复后的身份对账和封面可再生成性。
- 对路径、ZIP、鉴权、会话、限流、日志脱敏、队列背压和资源释放进行集中安全回归。
- 补齐用户部署、首次登录、APIkey 配置、备份恢复和升级文档。

验收方式：

- `scripts/acceptance/stage-08.sh` 从空卷安装、升级前一迁移版本、重建容器并执行冷备份恢复。
- 扫描进行中 Docker 仍健康；Content 不可访问时服务存活且不推断删除。
- 密码修改撤销全部会话，退出立即撤销当前会话，APIkey 轮换立即拒绝后续旧 key 请求。
- 自动安全测试证明不能越出 Content、跟随内部符号链接、覆盖冲突目标或通过日志获取秘密。
- 反复启动、取消图片请求和任务恢复后，句柄、线程、工作队列及临时文件不持续增长。

### 阶段 9：规模、性能与首版发布验收

实施内容：

- 固化 100,000 条代表性元数据生成器、真实 ZIP 图片基准、前后台混合负载和 Android 长时阅读脚本。
- 输出包含硬件、网络、存储、数据集、版本及冷热缓存条件的验收报告。
- 只修复阻断 PRD 指标的问题；任何指标调整必须评审并修改 PRD，不能静默降低。

验收方式：

- `scripts/acceptance/release.sh` 依次运行全部阶段回归、性能矩阵和稳定性测试。
- 空闲状态下，列表/搜索/筛选/排序 API 的 P95 ≤500ms；已有封面时 Web/Android 首屏 ≤1s。
- 预览图 1s 内开始返回且代表性请求 P95 ≤2s；未缓存原图 1s 内开始返回且 P95 ≤3s；Android 热缓存翻页 ≤300ms。
- 完整扫描 100,000 条元数据 ≤10 分钟；扫描期间前台 P95 不超过空闲指标两倍，评分确认 ≤1s。
- 服务空闲 RSS ≤300MB、60 分钟峰值 ≤1.5GB、最后十分钟稳定值不比最初十分钟高 10%；句柄、线程和队列无持续增长。
- Android 连续阅读 60 分钟无 OOM、持续明显掉帧或位图内存无界增长，并覆盖接近 20GB 缓存、清空缓存和网络恢复。
- 201 个真实 ZIP 的只读回归通过，最终 Docker 镜像、Web 静态资源和 Android APK 均可复现构建。

## 测试与交付规则

- Server：领域单元测试、SQLite 集成测试、临时文件系统测试、API 合约测试和负载测试分层运行。
- Web：组件测试加 Playwright E2E；所有管理测试使用一次性 Content 副本。
- Android：JVM、Compose、仪器测试和 60 分钟设备稳定性测试；真实服务端参与端到端验收。
- 每阶段提交验收报告，记录命令、版本、数据集、通过结果和已知限制；失败阶段不得以跳过测试作为完成。
- 阶段 0–3 建立服务端可用核心，阶段 4–5 完成 Web 管理闭环，阶段 6–7 完成 Android 阅读闭环，阶段 8–9 达到可发布标准。

## 明确假设与范围边界

- 计划覆盖三份 PRD 的完整首版，不另行裁剪 MVP。
- 本地开发默认将现有 `./content` 只读映射为容器 Content；测试写操作全部使用临时目录。
- 首版仅支持单用户、简体中文、标准未加密 ZIP、JPEG/PNG、在线连接服务端和 Android；不支持 Apple、完整离线书库、多用户、外网下载、手动 tag、上传、系列/章节、推荐或插件。
- Web 和 Android 串行建立在同一冻结 OpenAPI 语义之上；Web 先完成以便在开发 Android 前验证扫描、管理和图片链路。
- 浏览进度仅存 Android 本地；评分只存服务端一份；Web 不实现最近阅读或阅读进度。
